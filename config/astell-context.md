## What Astell is

Intelligence layer over a company's tools (astell.space, by Lab24). Ingests connected SaaS (Slack, Gmail, Notion, Linear, GitHub, Salesforce, HubSpot, Zendesk, +25 more), builds a cross-tool graph, surfaces open loops: decisions, owners, outcomes, drift. Stage: early access / waitlist. Team: small, pre-scale. Tagline: "Put context to work."

## The four pillars (and their current competitive status)

1. **Ingest, not retrieve** — reads tools in advance, in full. Status: contested; every serious player ingests now.
2. **Graph, not snapshot** — links every relationship across tools. Status: THE defensible bet. Nobody bundled has cracked decisions/owners/drift/open-loops.
3. **Cite, not hallucinate** — every line cites its source. Status: becoming table stakes.
4. **Mirror, not break** — respects source permissions. Status: commoditized (Credal sells it standalone; Notion/Dash/Copilot advertise it). No longer a differentiator alone.

## Standing strategic reads (update when they change)

- **Consolidation window is short**: 11+ category acquisitions in ~14 months; every suite is buying a context layer. Independence requires the cross-suite graph suites can't bundle.
- **Retrieval is commoditizing; the proactive graph isn't.** Astell's wire is open-loop/drift detection, not search.
- **Four vocabularies, one promise**: work-tool (Glean), data (Tanderrum/Omni), agent (Dust/Coworker), memory (Granola post-pivot/Cognee). Watch for cross-vocabulary pivots.
