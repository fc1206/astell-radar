Decision digest — what it means, what to do

2026-06-25 — digest (github)


1. Two un-tracked Tier 1s with marquee regulated logos expose a sovereignty wedge Astell's cloud model can't serve

Signal: This run registered two in-market direct competitors that pre-dated the baseline yet sat untracked, both selling cross-SaaS enterprise search but on a deployment model Astell lacks. Atolio (atolio.com) indexes Slack/Drive/Confluence/Jira with document-level permissions entirely inside the customer's own VPC/GovCloud/on-prem — "no data leaves your environment" — with the US Air Force, Cengage and Cribl as named customers and $24M Series A (Sep 2025, IBM Ventures + Bloomberg Beta). Squirro (squirro.com) sells GenAI enterprise search + knowledge graph over internal knowledge to central banks and regulated finance (ECB, Bank of England, HMRC, Standard Chartered), leaning on "deterministic," governed AI.
Why it matters to Astell: Both are direct-lane (pillar 2, cross-tool context) but win on a different axis than the Glean/Coworker/Adapt cluster: data residency and regulated-buyer trust. Astell's architecture — ingesting connected cloud SaaS into Astell — structurally cannot serve a buyer whose requirement is "nothing leaves our environment." That is an ICP-boundary question, not just another competitor: defense, government and central-bank buyers are a segment Astell either addresses with a deployment option or explicitly cedes.
Action: Write a one-page ICP-boundary memo by 2026-07-09 deciding whether Astell offers a private/in-VPC deployment tier or formally cedes the sovereignty segment (defense, regulated finance) to Atolio/Squirro — and add the resulting one-line qualifier ("cloud-native, not in-VPC") to the sales script and astell.space so reps disqualify those deals fast instead of losing them late.

2. Databricks Genie's new Managed MCP Server makes warehouse context agent-queryable — settle consume-vs-build with a spike

Signal: AI/BI Genie reached GA on all clouds and shipped a Managed MCP Server (beta) that lets any agent ask natural-language questions across Genie Spaces + Unity Catalog data, plus Genie Deep Research for multi-step analysis (https://www.databricks.com/blog/aibi-genie-now-generally-available). It lands the same week Snowflake pushed its own agentic-control-plane expansion and an OpenAI partnership (snowflake.com), with Summit recaps reporting it renamed "Snowflake Intelligence" to "Snowflake CoWork" — the data lane reaching for both MCP exposure and work-AI vocabulary at once.
Why it matters to Astell: Prior digests flagged data (WisdomAI, Tanderrum) as a vocabulary annexing Astell's promise; the new evidence is mechanical, not narrative — warehouse context is now a standardized MCP source any agent can read. That mirrors the consume-vs-compete calls already opened for Work IQ and Teamwork Graph: Astell could read warehouse-resident context via Genie's MCP server instead of building native warehouse connectors. Snowflake's "CoWork" naming is a parallel vocabulary land-grab from a distribution giant.
Action: Run a one-day eng spike by 2026-07-08 pointing Astell at Databricks Genie's Managed MCP Server to test reading warehouse context as an MCP source, and output an adopt/ignore note on consuming warehouse-as-context via MCP versus building native warehouse connectors — and confirm the Snowflake "CoWork" rename against a primary source before it forces a naming response.

Current threat assessment — top 5

  1. Glean — $300M ARR, $7.2B, openly using "context graph" language, now competing on cost-of-AI too. The incumbent startup in the lane.
  2. The incumbent bundle wave — M365 Copilot Work IQ, Gemini Enterprise, Slackbot/Agentforce, Rovo, Notion Enterprise Search, Dash. Permission-mirrored cross-SaaS search is becoming a *bundled feature*. Astell's wire is the proactive graph (open loops, decisions, drift), not retrieval — retrieval is commoditizing fast.
  3. Granola — $1.5B, pivoting from meeting notes to "enterprise context layer" with APIs + MCP. Converging on Astell from the meeting wedge, with beloved-product distribution. (We use it internally — know thy enemy.)
  4. OpenAI Workspace Agents — always-on agents over 60+ work apps with write actions. The "continuous, not one-shot" framing is a direct move toward open-loop detection.
  5. Coworker.ai (Village Platforms) — OM2 org knowledge graph, 50+ read-write connectors, aggressive pricing story, vs-Glean/vs-Granola/vs-Dust SEO pages, real customer logos. The most complete seed-stage clone of the full Astell thesis. (Homepage verified 2026-06-10.)
